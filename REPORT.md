# CS2 Faceit Stats Bot — Отчёт о разработке

## Зачем

Faceit показывает кучу цифр после матча: K/D, ADR, HS%, MVPs, триплы... Но **ничего не говорит по существу**. 

1.22 K/D — это хорошо? А если это саппорт с 80% KAST? А если entry-фрагер с 35% winrate открывашек?

Бот отвечает на один вопрос: **«Насколько игрок был полезен команде?»** — одной цифрой (−2.0 до +2.0) и вердиктом (Carry / Good / Average / Weak / Throw).

---

## Что делали

### Версия 0.1 — просто парсинг цифр

Самое начало: бот принимал скриншот, распознавал текст через Tesseract OCR, выдирал цифры и выдавал их обратно. Говно, потому что:
- OCR тупил на нестандартных шрифтах
- Faceit обновлял дизайн — всё ломалось
- Смысла ноль, просто пересказ того, что уже видно на экране

**Вывод:** пошли в API.

### Версия 0.2 — Faceit API + скоринг

Подключили Faceit API (`open.faceit.com/data/v4`). Появились:
- FaceitClient — запросы к API
- Агрегация данных матча
- Первая версия Usefulness Score

**Проблема:** скор считался ступеньками. K/D 1.2 и 1.4 давали 1.0, потому что оба попадали в `good`. А K/D 1.19 — уже 0.0 (average). Любой хороший игрок получал ~0.5, топы — 1.0.

**Фикс:** `_score_metric` — линейная интерполяция между порогами. Теперь K/D 1.35 → 1.5 (плавно между good и excellent).

### Версия 0.3 — MVPs и multi-kills

Добавили Impact-категорию (MVPs + мультикиллы). Без неё скор считался только по боёвке.

**Новая проблема:** Faceit API не отдаёт MVPs в match stats. Вообще. Никогда. 0. Всегда. Это ломало Impact — даже топ-игрок с 3+ триплами получал -1.0 по MVPs.

### Версия 0.4 — Faceit Analyser API (попытка)

Познакомились с Filipe (f1lipmeister), создателем Faceit Analyser. Он дал API-ключ. Задумка была:
- Получать расширенную статистику по матчу: entry/ trade/ utility/ clutch/ HLTV
- Использовать это для более точного скора (6 категорий вместо 2)

**Что пошло не так:** Cloudflare на стороне FA блокировал Render (наш хостинг). Render — это облачный провайдер, Cloudflare считает такие IP-диапазоны подозрительными. На локальной машине ключ работал идеально, с Render — 403 Forbidden.

Пытались:
- Менять endpoint `/api/match/...` на `/api/stats/...`
- Переключаться между query-параметром и заголовком
- Ставить fallback: сначала player-stats, потом match

Player-stats endpoint работал c curl, но с Render всё равно 403. **Победить Cloudflare не смогли** — он не пускал Render никак.

**Вывод:** FA отложили. Код интеграции удалён. Если захотим вернуться — нужен или свой прокси-сервер, или договариваться с Filipe о белом IP.

### Версия 0.5 — пересборка скора без FA

Посмотрели, что Faceit API и так даёт бесплатно:
- First Kills / First Deaths → **Entry** (процент выигранных открывашек)
- KAST — процент раундов, где игрок не был бесполезен
- MVPs + мультикиллы → **Impact**
- K/D, ADR, KPR, HS% → **Combat**

Собрали 4 категории с весами:
- Combat 35%
- Impact 20%
- Entry 25%
- KAST 20%

Без всяких внешних API. Только то, что Faceit сам отдаёт.


**Хостинг:** Render (free web service), авто-деплой при пуше в master.

**Стек:** Python 3.11, aiogram 3, httpx.

## Usefulness Score — как работает

Каждая метрика (K/D, ADR, entry%, KAST...) превращается в оценку от −2.0 до +2.0 через `_score_metric`:

```
excellent → 2.0
good      → 1.0–2.0 (линейно)
average   → 0.0–1.0 (линейно)
bad       → −1.0–0.0 (линейно)
ниже bad  → −2.0–−1.0
```

Пороги для каждого свои:
- K/D: excellent 1.5, good 1.2, average 0.9, bad 0.6
- Entry%: excellent 75%, good 60%, average 40%, bad 20%
- KAST: excellent 80%, good 70%, average 55%, bad 40%
- MVPs: excellent 4, good 2, average 0, bad −1 (0 не штрафует)

4 категории складываются с весами → общий скор.

Вердикты:
- ≥ 1.5 → Carry
- ≥ 0.5 → Good
- ≥ −0.5 → Average
- ≥ −1.5 → Weak
- < −1.5 → Throw

## Тесты

29 сценариев:
- **Carry:** топ-фраггер, donk-подобный импакт, абсолютный максимум
- **Good:** стабильный игрок, entry-фрагер, саппорт
- **Average:** ровный средний, чуть выше, чуть ниже
- **Weak:** дно таблицы, провальный entry
- **Throw:** полный провал 3-18, дисконнект
- **Edge:** 0 MVPs не штрафует, нет entry/KAST данных, монотонность
- **Consistency:** лучшие статы = выше скор, KAST спасает при плохих фрагах
